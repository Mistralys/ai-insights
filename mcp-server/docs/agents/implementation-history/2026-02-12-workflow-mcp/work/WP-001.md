# WP-001: Project Scaffolding

## Wave
Wave 1 -- Foundation + Read Tools

## Description
Set up the `mcp-server/` project from scratch at the repository root. This includes the Node.js package manifest, TypeScript configuration, the MCP server entry point that wires up `McpServer` with `StdioServerTransport`, and two small utility modules (timestamp formatting and work-package ID formatting/parsing). After this work package, `npx tsx mcp-server/src/index.ts` must start without errors (though no tools are registered yet).

## Requirements

1. **package.json** at `mcp-server/package.json`
   - `"name": "project-ledger-mcp"`
   - `"type": "module"`
   - Dependencies: `@modelcontextprotocol/sdk`, `zod`, `proper-lockfile`
   - Dev dependencies: `vitest`, `tsx`, `typescript`, `@types/node`, `@types/proper-lockfile`
   - Scripts: `"dev": "tsx src/index.ts"`, `"test": "vitest run"`, `"test:watch": "vitest"`

2. **tsconfig.json** at `mcp-server/tsconfig.json`
   - Target: `ES2022`, Module: `Node16` (or `NodeNext`), strict mode enabled
   - `outDir: "./dist"`, `rootDir: "./src"`
   - Include `src/**/*`

3. **Entry point** at `mcp-server/src/index.ts`
   - Import `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js`
   - Import `StdioServerTransport` from `@modelcontextprotocol/sdk/server/stdio.js`
   - Create a server instance with name `"project-ledger"` and version `"1.0.0"`
   - Connect to STDIO transport
   - Log startup message to stderr (stdout is reserved for MCP protocol)
   - No tools registered yet -- tools will be added in later WPs

4. **Utility: timestamp** at `mcp-server/src/utils/timestamp.ts`
   - Export function `now(): string` returning current time formatted as `"YYYY-MM-DD HH:MM:SS"` (matching the ledger schema format)

5. **Utility: wp-id** at `mcp-server/src/utils/wp-id.ts`
   - Export function `formatWpId(n: number): string` returning `"WP-###"` zero-padded to 3 digits
   - Export function `parseWpId(id: string): number` extracting the numeric part, throwing on invalid format

6. **Directory structure** must match the plan:
   ```
   mcp-server/
   ├── package.json
   ├── tsconfig.json
   └── src/
       ├── index.ts
       └── utils/
           ├── timestamp.ts
           └── wp-id.ts
   ```

7. **Install dependencies** after creating package.json (run `npm install` inside `mcp-server/`).

## Technical Constraints
- Transport is STDIO only -- no HTTP/SSE.
- All imports must use `.js` extensions for ESM compatibility with Node16 module resolution.
- Never write to stdout from application code; stdout is the MCP protocol channel.

## Acceptance Criteria
- [ ] `npm install` completes without errors inside `mcp-server/`.
- [ ] `npx tsx mcp-server/src/index.ts` starts the MCP server without runtime errors (verify by checking stderr output).
- [ ] `timestamp.now()` returns a string matching the pattern `YYYY-MM-DD HH:MM:SS`.
- [ ] `formatWpId(1)` returns `"WP-001"` and `parseWpId("WP-042")` returns `42`.
- [ ] TypeScript compiles without errors (`npx tsc --noEmit` in `mcp-server/`).

## Dependencies
None -- this is the foundational work package.
