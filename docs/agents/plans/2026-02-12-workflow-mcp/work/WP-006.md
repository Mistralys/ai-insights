# WP-006: Pipeline and Observation Tools

## Wave
Wave 3 -- Pipeline & Comments

## Description
Implement MCP tools for managing pipeline stages within work packages and for adding comments at both the pipeline and project levels. These tools handle the complex nested JSON structures that are most error-prone when agents manipulate files directly.

## Requirements

1. **start_pipeline** in `mcp-server/src/tools/pipeline.ts`
   - Input: `{ project_path: string, work_package_id: string, type: string }`
   - Behavior:
     - Validates WP is in IN_PROGRESS status
     - Checks no existing pipeline of the same type is already IN_PROGRESS (rejects duplicates)
     - Appends a new pipeline entry to the WP's `pipelines` array: `type`, `status: "IN_PROGRESS"`, `started_at` (now), empty `summary` array
     - Updates `last_updated` in root index
     - Uses `updateWorkPackageWithSync` for atomic writes
   - Error cases: WP not IN_PROGRESS, duplicate in-progress pipeline of same type

2. **complete_pipeline** in `mcp-server/src/tools/pipeline.ts`
   - Input: `{ project_path: string, work_package_id: string, type: string, status: "PASS" | "FAIL", summary: string[], artifacts?: object, metrics?: object, comments?: array, acceptance_criteria_updates?: { criterion: string, met: boolean }[] }`
   - Behavior:
     - Finds the most recent IN_PROGRESS pipeline of the given type
     - Sets `status` to PASS or FAIL
     - Sets `completed_at` to now
     - Sets `summary`, and optionally `artifacts`, `metrics`, `comments`
     - If `acceptance_criteria_updates` provided, updates matching criteria in the WP's `acceptance_criteria` array
     - Updates `last_updated` in root index
     - Uses `updateWorkPackageWithSync` for atomic writes
   - Error cases: no IN_PROGRESS pipeline of given type found, invalid status value

3. **add_observation** in `mcp-server/src/tools/observations.ts`
   - Input: `{ project_path: string, work_package_id: string, pipeline_type: string, type: string, priority: "low" | "medium" | "high", note: string }`
   - Behavior:
     - Finds the most recent pipeline of the given type (any status)
     - Appends a comment object to that pipeline's `comments` array: `type`, `priority`, `timestamp` (now), `note`
     - Pipeline comments do NOT include `agent` (agent is inferred from pipeline type)
     - Updates `last_updated` in root index
     - Uses `updateWorkPackageWithSync` for atomic writes
   - Error cases: WP not found, no pipeline of given type exists

4. **add_project_comment** in `mcp-server/src/tools/observations.ts`
   - Input: `{ project_path: string, type: string, priority: "low" | "medium" | "high", agent: string, note: string, context?: { os: string, tool: string, work_package?: string, resolved: boolean, workaround?: string } }`
   - Behavior:
     - Reads root index
     - Appends to `project_comments` array: `type`, `priority`, `timestamp` (now), `agent`, `note`
     - If type is `incident`, the `context` field is required and included
     - Updates `last_updated`
     - Uses file lock for root index write
   - Error cases: root index missing, `incident` type without `context`

## Technical Constraints
- Pipeline operations target the WP detail file but must also update `last_updated` in the root index.
- `complete_pipeline` must find the correct pipeline entry by matching both `type` and `status: "IN_PROGRESS"` (there could be multiple pipelines of the same type from revisions).
- Comment timestamps are auto-generated server-side (agents do not provide them).

## Acceptance Criteria
- [ ] `start_pipeline` appends a new IN_PROGRESS pipeline entry.
- [ ] `start_pipeline` rejects when WP is not IN_PROGRESS.
- [ ] `start_pipeline` rejects duplicate in-progress pipeline of the same type.
- [ ] `complete_pipeline` sets status, completed_at, summary, and optional fields.
- [ ] `complete_pipeline` updates acceptance criteria when `acceptance_criteria_updates` provided.
- [ ] `add_observation` appends a comment to the correct pipeline's comments array.
- [ ] `add_observation` does not include an `agent` field in the comment.
- [ ] `add_project_comment` appends to root index `project_comments`.
- [ ] `add_project_comment` requires `context` for `incident` type comments.
- [ ] All tools auto-generate timestamps.
- [ ] TypeScript compiles without errors.
