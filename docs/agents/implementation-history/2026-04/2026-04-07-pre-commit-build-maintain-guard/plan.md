# Plan: Pre-Commit Build & Maintain Guard

## Summary

Add automated pre-commit checks that remind and/or block the developer when version sync or CTX documentation is stale. Version mismatches (changelog vs package manifest) block the commit. Stale CTX documentation triggers a warning. Also fix stale "(gitignored)" references to `.context/` in documentation — the directory is tracked in VCS.

## Architectural Context

- **Pre-commit hook:** `.githooks/pre-commit` (shell script). Currently runs three checks: persona freshness (`node scripts/build-personas.js --check`), ruff linting on staged Python files, and a changelog drift warning. Activated via `git config core.hooksPath .githooks` (set up by `scripts/install-hooks.js`).
- **"Build & maintain" CLI command:** `scripts/cli.js` → `cmdBuildMaintain()`. Runs five sequential operations: sync MCP server version, sync orchestrator version, build personas, check role parity, regenerate CTX docs.
- **Version sync sources of truth:** Each module's `changelog.md` header (`## v{X.Y.Z}`) is the canonical version. Target files: `mcp-server/package.json`, `orchestrator/pyproject.toml`, `personas/package.json`.
- **CTX output:** `.context/` directory — tracked in VCS (not gitignored). Generated by `ctx generate` from `context.yaml` + `**/module-context.yaml` configs.
- **Stale docs:** `AGENTS.md` (lines 158, 359), `docs/agents/project-manifest/README.md` (line 115) incorrectly state `.context/` is gitignored. `CLAUDE.md` and `.context/agents.md` are auto-regenerated mirrors.

## Approach / Architecture

### 1. New Node.js script: `scripts/check-version-sync.js`

A standalone script that compares each module's changelog version against its package manifest version. Exits with code 1 on any mismatch, printing which modules are out of sync and how to fix it. This follows the existing pattern of `build-personas.js --check` — a dedicated Node.js check script called from the shell hook.

**Checks performed:**

| Module | Changelog | Package Manifest | Version Regex |
|--------|-----------|-----------------|---------------|
| MCP Server | `mcp-server/changelog.md` | `mcp-server/package.json` → `.version` | `^## v(\d+\.\d+\.\d+)` |
| Orchestrator | `orchestrator/changelog.md` | `orchestrator/pyproject.toml` → `version = "..."` | `^## v(\d+\.\d+\.\d+)` |
| Personas | `personas/changelog.md` | `personas/package.json` → `.version` | `^## v(\d+\.\d+\.\d+)` |

### 2. CTX staleness heuristic in the pre-commit hook

A shell-level check in `.githooks/pre-commit` that warns when source files are staged but no `.context/` files are staged alongside them. The heuristic: if any file matching `**/*.ts`, `**/*.py`, `**/*.js`, `**/*.yaml`, `**/*.md` under `mcp-server/src/`, `orchestrator/src/`, `personas/`, `scripts/`, or `shared/` is staged, **and** no file under `.context/` is staged, print a warning suggesting the developer run `node scripts/cli.js build-maintain` (or at minimum `node scripts/cli.js ctx-generate`). This is advisory only — does not block the commit.

### 3. Fix stale "(gitignored)" documentation

Remove the incorrect "(gitignored)" annotation from `AGENTS.md` and `docs/agents/project-manifest/README.md`. The `CLAUDE.md` and `.context/` copies will self-correct on the next build/CTX regeneration.

### 4. Extend the pre-commit hook

Add two new check blocks to `.githooks/pre-commit`:
1. **Version sync check (blocking):** `node scripts/check-version-sync.js || exit 1`
2. **CTX staleness check (warning):** Shell heuristic comparing staged source files vs staged `.context/` files.

Order in the hook after changes:
1. Persona freshness check (existing, blocking)
2. **Version sync check (new, blocking)**
3. Ruff linting (existing, blocking)
4. **CTX staleness warning (new, non-blocking)**
5. Changelog drift warning (existing, non-blocking)

## Rationale

- **Dedicated script vs inline shell:** Version extraction requires reading JSON and TOML files with regex. Node.js is already used for all other check scripts in the workspace. A standalone script is testable and reusable (can also be called from CI or the CLI).
- **CTX as warning, not blocker:** CTX regeneration takes time and requires the `ctx` binary on PATH. Blocking would be disruptive. A reminder is sufficient — the developer can choose to run it.
- **Heuristic approach for CTX:** Comparing file timestamps or checksums would be complex and fragile. Checking whether `.context/` files were staged alongside source changes is a simple, low-false-positive signal that works with the existing git workflow.

## Detailed Steps

### Step 1: Create `scripts/check-version-sync.js`

Create a new CJS Node.js script (matching the workspace convention) that:
1. Read each module's `changelog.md` and extract the first `## v{X.Y.Z}` version.
2. Read each module's package manifest (`package.json` or `pyproject.toml`) and extract the current version.
3. Compare. If any mismatch, print a clear error message with the remediation command and exit 1.
4. If all match, print a success line and exit 0.

The remediation message should suggest: `node scripts/cli.js build-maintain`

### Step 2: Extend `.githooks/pre-commit`

Add the version sync check (blocking) after the persona freshness check. Add the CTX staleness heuristic (warning) before the existing changelog drift warning.

The CTX staleness heuristic in shell:
```sh
# Detect staged source changes without corresponding .context/ updates
STAGED=$(git diff --cached --name-only)
SRC_CHANGED=$(echo "$STAGED" | grep -cE '^(mcp-server/src/|orchestrator/src/|personas/|scripts/|shared/)' || true)
CTX_CHANGED=$(echo "$STAGED" | grep -c '^\.context/' || true)

if [ "$SRC_CHANGED" -gt 0 ] && [ "$CTX_CHANGED" -eq 0 ]; then
  echo ""
  echo "WARNING: Source files changed but .context/ was not updated."
  echo "  Consider running: node scripts/cli.js build-maintain"
  echo ""
fi
```

### Step 3: Fix stale documentation

Edit `AGENTS.md`:
- Line 158: Change `(gitignored)` to `(tracked in VCS)` in the CTX Generator paragraph.
- Line 359: Remove `(gitignored)` from the `context.yaml` table row.

Edit `docs/agents/project-manifest/README.md`:
- Line 115: Change `gitignored` to `tracked in VCS`.

The downstream generated files (`CLAUDE.md`, `.context/agents.md`, `.context/project-manifest.md`) will be corrected automatically by the next `build-maintain` run.

### Step 4: Add CLI integration (optional enhancement)

Register `check-version-sync.js` as a standalone CLI command in `scripts/cli.js` so it can be invoked manually via `node scripts/cli.js check-versions`. This is a minor convenience — the script can already be run directly.

## Dependencies

- `node` must be available in the hook execution environment (already required by persona check).
- `ctx` binary is NOT required by the hook — the CTX check only warns about staleness, it doesn't regenerate.

## Required Components

- **New file:** `scripts/check-version-sync.js` — Version drift checker.
- **Modified file:** `.githooks/pre-commit` — Add version sync and CTX staleness checks.
- **Modified file:** `AGENTS.md` — Fix 2 stale "(gitignored)" references.
- **Modified file:** `docs/agents/project-manifest/README.md` — Fix 1 stale "gitignored" reference.

## Assumptions

- The `## v{X.Y.Z}` format in changelog headers is stable across all three modules (verified in codebase).
- The `pyproject.toml` version line format is `version = "{X.Y.Z}"` (verified in codebase).
- The pre-commit hook runs with the workspace root as CWD (standard git behavior).

## Constraints

- The hook must remain cross-platform (shell script). The version check delegates to Node.js (cross-platform). The CTX heuristic uses only POSIX-compatible shell commands (`grep -c`, `echo`).
- The script must use CJS (CommonJS) to match the `scripts/` directory convention, or be invoked via `node` without requiring ESM resolution.
- No new npm dependencies.

## Out of Scope

- Automatic version syncing in the hook (the hook detects drift; the developer runs `build-maintain` to fix it).
- Automatic CTX regeneration in the hook (too slow for a pre-commit hook).
- Fixing ledger dialogue files that mention "(gitignored)" — these are historical records.
- Syncing `personas/ledger/src/meta/_shared.yaml` → `default_version` (this is a template variable for generated persona files, separate from the build system version).

## Acceptance Criteria

- Committing with a changelog version that doesn't match the package manifest version is **blocked** with a clear error message naming the mismatched module(s) and the fix command.
- Committing source file changes without corresponding `.context/` updates produces a **warning** (non-blocking).
- All existing pre-commit checks (persona freshness, ruff, changelog drift) continue to work.
- `node scripts/check-version-sync.js` can be run standalone and returns exit code 0 when versions are in sync, 1 otherwise.
- The "(gitignored)" references in `AGENTS.md` and `docs/agents/project-manifest/README.md` are corrected.
- The hook works on macOS, Linux, and Windows (Git Bash).

## Testing Strategy

- **Manual test (version mismatch):** Temporarily edit a module's `package.json` version to a wrong value, stage a file, attempt `git commit` → expect block with clear message. Revert.
- **Manual test (CTX staleness):** Stage a source file change without staging any `.context/` file → expect warning.
- **Manual test (all green):** Run `node scripts/cli.js build-maintain`, stage everything → expect no warnings or blocks.
- **Standalone script test:** Run `node scripts/check-version-sync.js` after a fresh `build-maintain` → expect exit 0 and success message.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| **False positive CTX warning on non-content changes** (e.g., test-only changes) | The warning is advisory only and clearly suggests the fix command. Developer can safely ignore it for test-only commits. The heuristic scope (`mcp-server/src/`, `orchestrator/src/`, etc.) excludes `tests/` directories. |
| **Hook slows down commits** | The version check reads 6 small files (3 changelogs + 3 manifests). Sub-second. The CTX check is pure `grep` on the staged file list. Negligible. |
| **Windows compatibility of shell heuristic** | `grep -c` and `echo` work in Git Bash (the shell used for git hooks on Windows). The blocking check delegates to Node.js which is fully cross-platform. |
| **Developer bypasses hook with `--no-verify`** | This is intentional — `--no-verify` is a deliberate escape hatch. The CI pipeline (if added later) can enforce the same checks. |
