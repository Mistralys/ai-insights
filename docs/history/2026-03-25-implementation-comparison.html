Problem Statement

Compare two independent implementations of the same plan produced by two different AI-assisted workflows:

  • Workflow — VS Code Ledger workflow (branch test-workflow, 7 WPs)
  • Claude Teams — Claude Teams (branch test-claude-teams, single commit)

Evaluate each against five criteria (C1–C5) and determine which implementation is superior.

Context & Constraints

  • Both branches share the same merge base (e8d131b).
  • Both implement the same five-fix plan across two codebases (MCP server TypeScript, Orchestrator Python).
  • Existing constants STAGE_TO_PIPELINE and WP_TERMINAL_STATUSES already existed in orchestrator/src/config.py at the base commit.
  • The MCP tool ledger_update_work_package_status requires a parameter named agent (not agent_role).
  • Pre-existing test_graph.py failures (9 tests, aiosqlite missing) are present on both branches and unrelated to this work.

Recommendation: Workflow is the clearly superior implementation

It is more complete across every criterion except C4 (tie) and one minor code-reuse point.

  • Zero functional bugs vs. two bugs in Claude Teams.
  • 40+ new tests with 0 new failures vs. 12 tests with 7 failures.
  • 8 documentation files updated vs. 0.
  • Thorough spec additions vs. minimal spec additions.
  • Full synthesis report documenting strategic recommendations.
  • No new dependencies (stdlib sqlite3) vs. unresolved aiosqlite dep.

The one improvement to take from Claude Teams: import STAGE_TO_PIPELINE from config.py instead of duplicating it as _STAGE_PIPELINE_TYPE.
